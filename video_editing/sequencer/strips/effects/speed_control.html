Speed Control¶

Speed Control time-warps the strip, making it play faster or slower than it normally would. A Global Speed less than 1.0 makes the strip play slower; greater than 1.0 makes it play faster. Playing faster means that some frames are skipped, and the strip will run out of frames before the end frame. When the strip runs out of frames to display, it will just keep repeating the last one; action will appear to freeze. To avoid this, position the next strip under the original at a point where you want the motion to continue.

Options¶

Stretch to Input Strip Length

Automatically calculates the Speed Factor based on the length of the input strip. So if you make a strip 1/2 the original size the sequence will play back at 2 times the speed.

Use as Speed

Calculates the scale value based on a Speed Factor.

Note

You will have to manually re-adjust the length of the strip accordingly.

Speed Factor

Multiplies the current speed of the sequence by this value. So, a value of 0.5 will make the sequence half as fast while 2 would make the sequence twice as fast.

Frame Number

Specifies a frame to remap the current frame to, for example, setting this value to 50 displays the 50th frame. This can then be manually keyframed to recreate the animation.

Scale to Length

Maps the frame range on a 0-1 scale. For example, using this and a Frame Number of 0.5 will select the frame halfway through the sequence.

Multiply Speed

A multiplier applied after all other calculations.

Examples¶

Creating a Slow-Motion Effect¶

Suppose you want to slow your strip down. You need to affect the speed of the video clip without affecting the overall frame rate. Select the clip and Add ‣ Effect ‣ Speed Control effect strip. Click to drop it and press N to get the Properties. Uncheck the Stretch to input strip length option in the Effect Strip section. Set the Speed factor to be the factor by which you want to adjust the speed. To cut the displayed speed by 50%, enter 0.5. Now, a 275-frame clip will play at half speed, and thus display only the first 137 frames.

If you want the remaining frames to show in slow motion after the first set is displayed, double the Length of the source strip (since effects strip bounds are controlled by their source strips). If you are using a speed factor other than 0.5 then use the formula:

new_length = real_length / speed_factor

That is it, set your render to animate (in this example) all 550 frames.

Keyframing the Speed Control¶

Keyframing the Frame number.¶

To get even finer control over your clip timing, you can use curves! While it is possible to keyframe the Speed factor, usually you want to keyframe the Frame number directly.

Uncheck Stretch to input strip length and uncheck Use as speed. You now have a Frame number field which you can keyframe. If you want the strip to animate at all you will have to insert some keyframes, otherwise it will look like a still. In most cases you will want to use the Graph editor view to set the curve interpolation to Linear since the default Bézier will rarely be what you want.

Tip

If you choose to keyframe the Speed factor instead, remember to Refresh All or the changes will not take effect.

Changing Video Frame Rates¶

You can use the speed control to change the frame rate in frames per second (fps) of a video. If you are rendering your video to a sequence set, you can effectively increase or decrease the number of individual image files created, by using a Global Speed value less than or greater than one, respectively. For example, if you captured a five-minute video at 30 fps and wanted to transfer that to film, which runs at 24 fps, you would enter a Global Speed of 30/24, or 1.25 (and Enable Frame Blending to give that film blur feel). Instead of producing 5 × 60 × 30 = 9000 frames, Blender would produce 9000 / 1.25 = 7200 = 5 × 60 × 24 frames. In this case, you set a start = 1 and end = 7200, set your Format output to jpeg 30fps, and image files 0001.jpg through 7200.jpg would be rendered out, but those images cover the entire 9000 frames. The image file 7200.jpg is the same at frame 9000. When you read those images back into your film blend-file at 24 fps, the strip will last exactly 5 minutes.

Kontrol Kecepatan ¶

Kontrol Kecepatan mengubah strip, membuatnya berputar lebih cepat atau lebih lambat dari biasanya. Kecepatan Global kurang dari 1.0 membuat pemutaran strip lebih lambat; lebih besar dari 1.0 membuatnya bermain lebih cepat. Bermain lebih cepat berarti beberapa frame dilewati, dan strip akan kehabisan bingkai sebelum bingkai akhir. Ketika strip kehabisan bingkai untuk ditampilkan, itu akan terus mengulangi yang terakhir; tindakan akan tampak membeku. Untuk menghindari ini, posisikan strip berikutnya di bawah aslinya pada titik di mana Anda ingin melanjutkan gerakan.

Opsi ¶

Regangkan ke Input Strip Length

Secara otomatis menghitung Faktor Kecepatan berdasarkan panjang strip masukan. Jadi jika Anda membuat strip 1/2 ukuran aslinya, urutan akan diputar ulang dengan kecepatan 2 kali lipat.

Gunakan sebagai Kecepatan

Menghitung nilai skala berdasarkan Faktor Kecepatan .

Catatan

Anda harus menyesuaikan ulang panjang strip secara manual.

Faktor Kecepatan

Mengalikan kecepatan urutan saat ini dengan nilai ini. Jadi, nilai 0,5 akan membuat barisan setengah lebih cepat sementara 2 akan membuat barisan dua kali secepat.

Nomor Bingkai

Menentukan bingkai untuk memetakan ulang bingkai saat ini, misalnya, menyetel nilai ini ke 50 akan menampilkan bingkai ke-50. Ini kemudian dapat secara manual keyframed menjadi membuat ulang animasinya.

Skalakan ke Panjang

Memetakan rentang bingkai pada skala 0-1. Misalnya, menggunakan ini dan Nomor Bingkai 0,5 akan memilih frame di tengah urutan.

Lipatgandakan Kecepatan

Pengganda diterapkan setelah semua penghitungan lainnya.

Contoh ¶

Membuat Efek Gerak Lambat ¶

Misalkan Anda ingin memperlambat strip Anda. Anda perlu memengaruhi kecepatan klip video tanpa memengaruhi keseluruhan kecepatan bingkai. Pilih klip dan strip efek Tambahkan ‣ Efek ‣ Kontrol Kecepatan . Klik untuk melepaskannya dan tekan N untuk mendapatkan Properti. Hapus centang opsi Regangkan ke masukan panjang strip di bagian Strip Efek. Setel Faktor kecepatan menjadi faktor yang Anda ingin sesuaikan kecepatannya. Untuk memotong kecepatan yang ditampilkan sebesar 50%, masukkan 0,5. Sekarang, klip 275 bingkai akan diputar dengan kecepatan setengah, dan karenanya hanya menampilkan 137 bingkai pertama.

Jika Anda ingin sisa frame ditampilkan dalam gerakan lambat setelah rangkaian pertama ditampilkan, menggandakan Panjang strip sumber (karena batas strip efek dikendalikan oleh strip sumbernya). Jika Anda menggunakan faktor kecepatan selain 0,5, gunakan rumus:

new_length = real_length / speed_factor

Selesai, setel render Anda untuk menganimasikan (dalam contoh ini) ke 550 frame.

Membingkai Utama Kontrol Kecepatan¶

Membuat bingkai utama nomor Bingkai. ¶

Untuk mendapatkan kontrol yang lebih baik atas waktu klip Anda, Anda dapat menggunakan kurva! Meskipun dimungkinkan untuk membingkai faktor Kecepatan, biasanya Anda ingin membingkai nomor Frame secara langsung.

Hapus centang Regangkan untuk memasukkan panjang setrip dan hapus centang Gunakan sebagai kecepatan . Anda sekarang memiliki bidang nomor Bingkai yang dapat Anda buat bingkai utama. Jika Anda ingin strip beranimasi sama sekali Anda harus memasukkan beberapa bingkai utama, jika tidak maka akan terlihat seperti diam. Dalam kebanyakan kasus, Anda mungkin ingin menggunakan tampilan editor Grafik untuk mengatur interpolasi kurva ke Linear karena Bézier default jarang menjadi yang Anda inginkan.

Kiat

Jika Anda memilih untuk membingkai faktor Kecepatan sebagai gantinya, ingatlah untuk Segarkan Semua atau perubahan tidak akan diterapkan.

Mengubah Rasio Bingkai Video ¶

Anda dapat menggunakan kontrol kecepatan untuk mengubah kecepatan frame dalam frame per detik (fps) video. Jika Anda merender video Anda ke rangkaian urutan, Anda dapat secara efektif menambah atau mengurangi jumlah file gambar individual yang dibuat, dengan menggunakan nilai Kecepatan Global masing-masing kurang dari atau lebih besar dari satu. Sebagai contoh, jika Anda merekam video lima menit dengan kecepatan 30 fps dan ingin mentransfernya ke film, yang berjalan pada 24 fps, Anda akan memasukkan Kecepatan Global 30/24, atau 1,25 (dan Aktifkan Pencampuran Bingkai untuk memberikan kesan buram pada film). Alih-alih menghasilkan 5 × 60 × 30 = 9000 < / kode> bingkai, Blender akan menghasilkan 9000 / 1,25 = 7200 = 5 < span class = "pre"> × 60 × 24 bingkai. Dalam kasus ini, Anda menyetel start = 1 dan end = 7200, setel keluaran Format Anda ke jpeg 30fps, dan file gambar 0001.jpg melalui 7200.jpg akan dirender di luar, tetapi gambar-gambar itu mencakup keseluruhan 9.000 bingkai. File gambar 7200.jpg sama di bingkai 9000. Saat Anda membaca gambar-gambar itu kembali ke file campuran film Anda pada 24 fps, strip akan bertahan tepat 5 menit.
