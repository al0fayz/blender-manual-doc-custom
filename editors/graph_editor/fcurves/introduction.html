Introduction¶

After animating some property in Blender using keyframes you can edit their corresponding curves. When something is “animated”, it changes over time. This curve is shown as something called an F-curve. Basically what an F-curve does is an interpolation between two animated properties. In Blender, animating an object means changing one of its properties, such as the object’s location, or its scale.

As mentioned, Blender’s fundamental unit of time is the “frame”, which usually lasts just a fraction of a second, depending on the frame rate of the scene. As animation is composed of incremental changes spanning multiple frames, usually these properties are not manually modified frame-by-frame, because:

  • It would take ages!

  • It would be very difficult to get smooth variations of the property (unless you compute mathematical functions and type a precise value for each frame, which would be crazy).

This is why nearly all direct animation is done using interpolation.

The idea is simple: you define a few Keyframes, which are multiple frames apart. Between these keyframes, the properties’ values are computed (interpolated) by Blender and filled in. Thus, the animators’ workload is significantly reduced.

Example of an interpolation.¶

For example, if you have:

  • A control point of value 0 at frame 0,

  • another one of value 10 at frame 25,

  • and you use linear interpolation, then, at frame 5 we get a value of 2.

The same goes for all intermediate frames: with just two points, you get a smooth increase from (0 to 10) along the 25 frames. Obviously, if you would like the frame 15 to have a value of 9, you would have to add another control point (or keyframe)…

Direction of Time¶

Although F-curves are very similar to Bézier, there are some important differences.

For obvious reasons, a property represented by a curve cannot have more than one value at a given time, hence:

  • When you move a control point ahead of a control point that was previously ahead of the point that you are moving, the two control points switch their order in the edited curve, to avoid the curve going back in time.

  • For the above reason, it is impossible to have a closed F-curve.

Settings¶

F-Curves have three additional properties, which control the interpolation between points, extension behavior, and the type of handles.

Interpolation Mode¶

Reference

Menu

Key ‣ Interpolation Mode

Hotkey

T

Mode for the Interpolation between the current and next keyframe.

Interpolation¶

Constant

There is no interpolation at all. The curve holds the value of its last keyframe, giving a discrete (stairway) “curve”. Usually only used during the initial “blocking” stage in pose-to-pose animation workflows.

Constant.¶

Linear

This simple interpolation creates a straight segment, giving a non-continuous line. It can be useful when using only two keyframes and the Extrapolation extend mode, to easily get an infinite straight line (i.e. a linear curve).

Linear.¶

Bézier

The more powerful and useful interpolation, and the default one. It gives nicely smoothed curves, i.e. smooth animations!

Bézier.¶

Note

Remember that some F-curves can only take discrete values, in which case they are always shown as if constant interpolated, whatever option you chose.

Easing (by strength)¶

Different methods of easing interpolations for F-curve segment. The “Robert Penner easing equations” (basically, equations which define some preset ways that one keyframe transitions to another) which reduce the amount of manual work (inserting and tweaking keyframes) to achieve certain common effects. For example, snappy movements.

  • Linear

  • Sinusoidal

  • Quadratic

  • Cubic

  • Quartic

  • Quintic

  • Exponential

  • Circular

See also

For more info and a few live demos, see http://easings.net and http://www.robertpenner.com/easing/

Dynamic Effects¶

These additional easing types imitate (fake) physics-based effects like bouncing/springing effects. The corresponding settings can be found in the Sidebar region ‣ Active Keyframe panel.

Elastic

Exponentially decaying sine wave, like an elastic band. This is like bending a stiff pole stuck to some surface, and watching it rebound and settle back to its original state.

Amplitude

The amplitude property controls how strongly the oscillation diverges from the basic curve. At 0.0, there is no oscillation (i.e. it just snaps to the B-value like an extreme exponential transition), and at 1.0 a profile similar to the one shown in the icon occurs.

Period

The period property controls the frequency with which oscillations occur. Higher values result in denser oscillations.

Bounce

Exponentially decaying parabolic bounce, like when objects collide. e.g. for Bouncing balls, etc.

Back

Cubic easing with overshoot and settle. Use this one when you want a bit of an overshoot coming into the next keyframe, or perhaps for some wind-up anticipation.

Back

The back property controls the size and direction (i.e. above/below the curve) of the overshoot.

Easing Type¶

Reference

Menu

Key ‣ Easing Type

Hotkey

Ctrl-E

The Easing Type controls which end of the segment between the two keyframes that the easing effects apply to.

Automatic Easing

The most commonly expected of the below behaviors is used. For the transitional effects, this is basically ease in, while for the physics effects it is ease out.

Ease In

Effect builds up to the second keyframe.

Ease Out

Effect fades out from the first keyframe.

Ease In Out

Effect occurs on both ends of the segment.

Extrapolation¶

Reference

Menu

Channel ‣ Extrapolation Mode

Hotkey

Shift-E

Extrapolation defines the behavior of a curve before the first and after the last keyframes.

There are two basic extrapolation modes:

Constant

The default one, curves before their first keyframe and after their last one have a constant value (the one of these first and last keyframes).

Constant extrapolation.¶

Linear

Curves ends are straight lines (linear), as defined by the slope of their first and last keyframes.

Linear extrapolation.¶

Additional extrapolation methods (e.g. the Cycles modifier) are located in the F-Curve Modifiers.

Handle Types¶

Reference

Menu

Key ‣ Handle Type

Hotkey

V

When using Bézier-interpolated curves it is possible to control the slope of the curve at the control points. This is done via the curve point handles; you can set the type of handle to use for the curve points by pressing V or choosing Key, Handle Type in the Graph editor menu. Each curve point can have a different handle type, even within the same curve.

There are three automatic modes, Automatic, Auto Clamped, and Vector, where Blender automatically determines the curve’s slope at each control point. The neighboring control points have the most influence of the slope, and points further away have a smaller influence. This can be controlled per F-curve with the Auto Handle Smoothing properties.

By using the other, non-automatic modes, you have full manual control over the slope.

Automatic

Handle positions are automatically chosen to produce smooth curves.

Auto handles.¶

Auto Clamped

Automatic handles clamped to prevent overshoots and changes in the curve direction between keyframes (S-shapes).

Auto clamped handles.¶

Vector

Creates automatic linear interpolation between keyframes. The segments remain linear when keyframe centers are moved. However, when the handles are moved, the handle type switches to Free.

Vector handles.¶

Aligned

The two handles of the curve point are locked together to always point in exactly opposite directions. This results in a curve that is always smooth at the control point.

Aligned handles.¶

Free

The handles can be moved completely independently, and thus can result in a sharp change of direction.

Free handles.¶

Pendahuluan ¶

Setelah menganimasikan beberapa properti di Blender menggunakan bingkai utama, Anda dapat mengedit kurva yang sesuai. Ketika sesuatu "dianimasikan", itu berubah seiring waktu. Kurva ini ditampilkan sebagai sesuatu yang disebut Kurva F. Pada dasarnya apa yang dilakukan kurva-F adalah interpolasi antara dua properti animasi. Di Blender, menganimasikan sebuah objek berarti mengubah salah satu propertinya, seperti lokasi objek, atau skalanya.

Seperti yang disebutkan, unit dasar waktu Blender adalah "bingkai", yang biasanya berlangsung hanya sepersekian detik, bergantung pada frekuensi gambar pemandangan. Karena animasi terdiri dari perubahan inkremental yang mencakup beberapa bingkai, biasanya properti ini tidak diubah frame-by-frame secara manual, karena:

  • Ini akan memakan waktu lama!

  • Akan sangat sulit untuk mendapatkan variasi properti yang mulus (kecuali Anda menghitung fungsi matematika dan mengetikkan nilai yang tepat untuk setiap bingkai, yaitu gila).

Inilah mengapa hampir semua animasi langsung dilakukan menggunakan interpolasi .

Idenya sederhana: Anda menentukan beberapa Bingkai Utama, yang merupakan beberapa bingkai terpisah. Di antara bingkai utama ini, nilai properti dihitung (diinterpolasi) oleh Blender dan diisi. Dengan demikian, beban kerja animator berkurang secara signifikan.

Contoh interpolasi. ¶

Misalnya, jika Anda memiliki:

  • Titik kontrol nilai 0 pada frame 0,

  • satu lagi dari nilai 10 pada frame 25,

  • dan Anda menggunakan interpolasi linier, maka pada frame 5 kita mendapatkan nilai 2.

Hal yang sama berlaku untuk semua bingkai perantara: dengan hanya dua titik, Anda mendapatkan peningkatan yang mulus dari (0 menjadi 10) sepanjang 25 bingkai. Jelas, jika Anda ingin bingkai 15 memiliki nilai 9, Anda harus menambahkan titik kontrol lain (atau bingkai utama)…

Arah Waktu ¶

Meskipun kurva-F sangat mirip dengan Bézier , ada beberapa perbedaan penting.

Untuk alasan yang jelas, properti diwakili oleh kurva tidak dapat memiliki lebih dari satu nilai pada waktu tertentu, karenanya:

  • Saat Anda memindahkan titik kontrol di depan titik kontrol yang sebelumnya berada di depan titik tersebut bahwa kamu pindah, dua titik kontrol mengganti urutannya dalam kurva yang diedit, untuk menghindari kurva masuk kembali waktu.

  • Karena alasan di atas, tidak mungkin kurva F tertutup.

Peralihan dua titik kontrol: kurva tidak dapat masuk kembali waktu!¶

Sebelum memindahkan bingkai utama kedua.¶

Setelah memindahkan bingkai utama kedua.¶

Setelan ¶

Kurva-F memiliki tiga properti tambahan, yang mengontrol interpolasi antar titik, perilaku ekstensi, dan jenis pegangan.

Mode Interpolasi ¶

Referensi

Menu

Kunci ‣ Mode Interpolasi

Hotkey

T

Mode untuk Interpolasi antara bingkai utama saat ini dan berikutnya.

Interpolasi ¶

Konstan

Tidak ada interpolasi sama sekali. Kurva menahan nilai keyframe terakhirnya, memberikan "kurva" diskrit (tangga). Biasanya hanya digunakan selama tahap "pemblokiran" awal dalam alur kerja animasi pose-ke-pose.

Konstan. ¶

Linear

Interpolasi sederhana ini membuat segmen lurus, menghasilkan garis yang tidak bersambung. Ini dapat berguna saat menggunakan hanya dua bingkai utama dan mode perluasan Ekstrapolasi , untuk dengan mudah mendapatkan garis lurus tak terbatas (yaitu kurva linier).

Linear. ¶

Bézier

Interpolasi yang lebih kuat dan berguna, dan yang standar. Ini memberikan kurva yang dihaluskan dengan baik, yaitu animasi yang halus!

Bézier. ¶

Catatan

Ingatlah bahwa beberapa kurva F hanya dapat mengambil nilai diskrit, dalam hal ini mereka selalu ditampilkan seolah-olah diinterpolasi konstan, opsi apa pun yang Anda pilih.

Easing (berdasarkan kekuatan) ¶

Berbagai metode easing interpolasi untuk segmen kurva F. The "Robert Penner easing persamaan" (pada dasarnya, persamaan yang mendefinisikan beberapa cara yang telah ditetapkan sebelumnya satu transisi bingkai utama ke yang lain) yang mengurangi jumlah pekerjaan manual (memasukkan dan mengutak-atik bingkai utama) untuk mencapai efek umum tertentu. Misalnya gerakan yang tajam.

  • Linear

  • Sinusoidal

  • Kuadrat

  • Kubik

  • Quartic

  • Quintic

  • Eksponensial

  • Melingkar

Lihat juga

Untuk info lebih lanjut dan beberapa demo langsung, lihat http://easings.net dan http://www.robertpenner.com/easing/

Efek Dinamis ¶

Jenis pelonggaran tambahan ini karena efek berbasis fisika (palsu) seperti memantul / meloncat efek. Pengaturan yang terkait dapat ditemukan di Wilayah bilah sisi ‣ Panel Keyframe aktif .

Elastis

Gelombang sinus yang membusuk secara eksponensial, seperti karet gelang. Ini seperti menekuk tiang kaku yang menempel pada suatu permukaan, dan menontonnya pulih dan kembali ke keadaan semula.

Amplitudo

Properti amplitudo kontroler kuat apapun osilasi menyimpang dari dasar melengkung. Pada 0,0, tidak ada osilasi (hanya mengatakan ke nilai-B seperti yang ekstrim berpindah-pindah eksponensial), dan pada 1.0 profil yang mirip dengan yang ditunjukkan pada ikon muncul.

Periode

Properti periode frekuensi kejadian osilasi. Nilai yang lebih tinggi menghasilkan osilasi yang lebih padat.

Pantulan

Pentalan parabola yang membusuk secara eksponensial, seperti saat objek bertabrakan. misalnya untuk bola memantul, dll.

Kembali

Pelonggaran kubik dengan overshoot dan penyelesaian. Gunakan yang ini saat Anda ingin sedikit overshoot pada keyframe berikutnya, atau mungkin untuk antisipasi penyelesaian akhir.

Kembali

Properti kembali mengontrol ukuran dan arah (yaitu di atas / di bawah kurva) dari melebihi batas.

Jenis Easing ¶

Referensi

Menu

Kunci ‣ Jenis Easing

Hotkey

Ctrl-E

Jenis Easing mengontrol akhir segmen antara dua bingkai utama yang easing efek berlaku untuk.

Easing Otomatis

Yang paling umum diharapkan dari perilaku di bawah ini digunakan. Untuk efek transisi, ini pada dasarnya adalah kemudahan dalam , sedangkan untuk fisika efeknya adalah memudahkan .

Kemudahan

Efek berkembang hingga bingkai utama kedua.

Ease Out

Efek menghilang dari bingkai utama pertama.

Ease In Out

Efek terjadi di kedua ujung segmen.

Ekstrapolasi ¶

Referensi

Menu

Saluran ‣ Mode Ekstrapolasi

Hotkey

Shift-E

Ekstrapolasi menentukan perilaku kurva sebelum bingkai utama pertama dan setelah bingkai utama terakhir.

Ada dua mode ekstrapolasi dasar:

Konstan

Bawaan, kurva sebelum bingkai utama pertama dan setelah bingkai terakhir memiliki konstanta nilai (salah satu dari bingkai utama pertama dan terakhir ini).

Ekstrapolasi konstan. ¶

Linear

Ujung kurva adalah garis lurus (linier), sebagaimana ditentukan oleh kemiringan garis pertama dan terakhirnya keyframes.

Ekstrapolasi linier. ¶

Metode ekstrapolasi tambahan (mis. pengubah Siklus ) berada di F-Curve Pengubah .

Jenis Pegangan ¶

Referensi

Menu

Kunci ‣ Jenis Pegangan

Hotkey

V

Saat menggunakan kurva Bézier-interpolated, dimungkinkan untuk mengontrol kemiringan kurva di titik kontrol. Ini dilakukan melalui pegangan titik kurva; Anda dapat mengatur jenis pegangan yang akan digunakan untuk titik kurva dengan menekan V atau memilih Key, Handle Type di menu Graph editor. Setiap titik kurva dapat memiliki jenis tuas yang berbeda, bahkan dalam kurva yang sama.

Ada tiga mode otomatis, Otomatis , Dijepit Otomatis , dan Vektor , di mana Blender secara otomatis menentukan kemiringan kurva di setiap titik kontrol. Titik kontrol tetangga memiliki pengaruh paling besar terhadap kemiringan, dan poin yang lebih jauh memiliki pengaruh yang lebih kecil. Ini dapat dikontrol per kurva-F dengan Auto Handle Smoothing .

Dengan menggunakan mode non-otomatis lainnya, Anda memiliki kontrol manual penuh atas kemiringan.

Otomatis

Posisi pegangan secara otomatis dipilih untuk menghasilkan kurva yang mulus.

Tuas otomatis. ¶

Dijepit Otomatis

Pegangan otomatis dijepit untuk mencegah overshoot dan perubahan arah kurva antara bingkai utama (bentuk-S).

Tuas yang dijepit otomatis. ¶

Vektor

Membuat interpolasi linier otomatis antara bingkai utama. Segmen tetap linier saat pusat bingkai utama dipindahkan. Namun, saat gagang dipindahkan, jenis pegangan beralih ke Gratis .

Tuas vektor. ¶

Diratakan

Kedua tuas titik kurva dikunci untuk selalu menunjuk tepat berlawanan petunjuk arah. Ini menghasilkan kurva yang selalu mulus pada titik kontrol.

Tuas disejajarkan. ¶

Gratis

Pegangan dapat dipindahkan sepenuhnya secara terpisah, dan dengan demikian dapat menghasilkan perubahan tajam pada arah.

Pegangan gratis.¶
